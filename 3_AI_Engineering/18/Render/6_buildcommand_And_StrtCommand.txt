## 🛠️ Build vs Start in Render
- **Build Command**
  - Runs **once during deployment**.
  - Purpose: prepare the app for production.
  - Tasks: install dependencies, transpile/compile code (React → static files, TypeScript → JS), bundle assets.
  - Example:  
    ```bash
    npm ci --omit=dev && npm run build
    ```
  - Output: a production-ready build folder or compiled server code.

- **Start Command**
  - Runs **every time the service boots** (after build).
  - Purpose: launch the app using the already-built code.
  - Example:  
    ```bash
    npm start
    ```
  - It doesn’t rebuild — it just executes the server or serves static files.

👉 Analogy:  
- **Build** = baking the cake.  
- **Start** = serving the cake.  
You don’t re-bake every time; you just serve what’s already prepared.

---

## 📦 Why `npm ci --omit=dev`
- **`npm ci`**
  - Clean install based on `package-lock.json`.
  - Deletes `node_modules` and reinstalls exactly what’s locked.
  - Faster and more reliable than `npm install`.

- **`--omit=dev`**
  - Skips installing devDependencies (linters, test frameworks, build tools).
  - Keeps the container smaller, faster, and more secure.
  - Ensures only runtime dependencies are present in production.

---

## 🚀 Typical Render Workflow
1. **Build Command**  
   ```bash
   npm ci --omit=dev && npm run build
   ```
   - Installs only production dependencies.
   - Compiles/transpiles the app for deployment.

2. **Start Command**  
   ```bash
   npm start
   ```
   - Boots the server using the compiled code.

---

## 🧠 Key Insight
- **Build** prepares the app (production-ready code).  
- **Start** runs the app (serves that code).  
- **`npm ci --omit=dev`** ensures the build step is lean, reproducible, and safe for production by excluding unnecessary dev tools.
