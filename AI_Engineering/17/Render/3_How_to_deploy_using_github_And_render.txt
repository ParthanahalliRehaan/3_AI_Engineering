```bash
# 1. Initialize your Node.js project
mkdir my-app && cd my-app
npm init -y

# 2. Install dependencies (example: Express)
npm install express

# 3. Create index.js (entry point)
cat > index.js << 'EOF'
const express = require('express');
const app = express();
const PORT = process.env.PORT || 3000;

app.get('/', (req, res) => {
  res.send('Hello from Render + GitHub!');
});

app.listen(PORT, () => {
  console.log(`Server running on port ${PORT}`);
});
EOF

# 4. Update package.json scripts
# Open package.json and ensure you have:
# {
#   "scripts": {
#     "start": "node index.js"
#   }
# }

# 5. Create a .gitignore file
cat > .gitignore << 'EOF'
node_modules
.env
EOF

# 6. Initialize Git and push to GitHub
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/<your-username>/<your-repo>.git
git push -u origin main

# 7. Go to Render dashboard
# - Click "New Web Service"
# - Connect your GitHub repo
# - Select branch: main
# - Build command: npm install
# - Start command: npm start

# 8. Setup environment variables in Render
# In Render dashboard → Environment → Add variables
# Example:
#   KEY=VALUE
#   DATABASE_URL=postgres://...

# 9. Deploy
# Render will automatically build and deploy your app.
# Every push to main triggers a redeploy.
```

⚡ This gives you a **minimal working template**:  
- `index.js` → Express server  
- `package.json` → `start` script for Render  
- `.gitignore` → keeps secrets & node_modules out of GitHub  
- Render → handles build, env vars, hosting
