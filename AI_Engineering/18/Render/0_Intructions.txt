**Context**
The lesson is about deploying to render.com after uploading code to GitHub.

**Build Command & Start Command**
- Render's default build command is `npm install`
- Start command: `node server.js`
- These work fine, but there's room to optimize

**Dependencies vs Dev Dependencies**
- **Regular dependencies** — packages the app needs to actually run in production (e.g., Express, AI SDKs, database packages)
- **Dev dependencies** — only needed during development (e.g., Nodemon, TypeScript compilers, Prettier, ESLint, Jest, Vitest)
- Goal: keep the production build as light as possible by excluding dev dependencies

**Common Misconception about NODE_ENV**
- Render sets `NODE_ENV=production` at runtime, but this does **NOT** automatically omit dev dependencies — you have to do it yourself

**How to Omit Dev Dependencies**
- `npm install --omit=dev` — installs without dev dependencies
- `npm ci --omit=dev` — **better option**; CI = Clean Install, uses `package-lock.json` for a more reliable and reproducible build; best for deployment

**⚠️ Caution**
- Some packages often placed in dev dependencies are actually needed at build time — compilers and bundlers like Webpack, TSC, Vite, ESBuild. Don't accidentally omit these.

**Challenge in the Lesson**
1. Run `npm install` → `npm run dev` → app starts with Nodemon ✅
2. Stop server → `npm ci --omit=dev` → `npm run dev` → **Nodemon not found error** ✅
3. Run `npm start` → app runs normally, but without Nodemon ✅

**Render Dashboard Settings**
- Build command: `npm ci --omit=dev`
- Start command: `npm start` (cleaner than `node server.js`)
- Instance type: **Free tier** (fine for first deployment)

**Environment Variables**
- Two options on Render: paste multiple at once from your `.env` file, or add one by one
- Must add your API key (e.g., `OPENAI_API_KEY`) and optionally your AI model name
- Key names must match **exactly** what's used in your code

**Final Result**
- Click Deploy → wait a few seconds to minutes → site goes live with a render.com subdomain
